## Learnings

### 1. Parallel Agent Research Pattern
When exploring unknown territory, spawn N agents with different search vectors. Combine results. Much faster than serial exploration.

**When to use**: New repo onboarding, deep context gathering, history reconstruction.

### 2. Oracle Thread as Project Context
Thread 13 serves as the "canonical context" for voice-tray-v2. Any new session can query it:
```
oracle_thread(threadId: 13, message: "your question")
```
The thread accumulates knowledge across sessions.

### 3. Graduated Projects Need Their Own Soul
When a project spins off from Nat-s-Agents:
1. Create own Ïˆ/ (soul-lite is enough)
2. Install Oracle commands (/awaken)
3. Reference parent thread for history
4. Retrospectives stay local, learnings can flow back

### 4. DB Hunting Strategy
When MCP tool doesn't expose data directly:
1. Find the MCP server process (`ps aux | grep`)
2. Find source code (`/src/`)
3. Grep for DB_PATH or connection strings
4. Read schema, query directly with sqlite3
